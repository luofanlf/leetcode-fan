# 236.二叉树的最近公共祖先

https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/?envType=study-plan-v2&envId=top-100-liked

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        return findNode(root,p,q);
    }
    public TreeNode findNode(TreeNode node,TreeNode p,TreeNode q){
        if(node == null || node==p || node == q){
            return node;
        }
        TreeNode left = findNode(node.left,p,q);
        TreeNode right = findNode(node.right,p,q);
        
        if(left == null) {
            return right;
        }
        if(right == null){
            return left;
        }
        return root;
    }
}
```

对于二叉树的最近公共祖先的定义，最近公共祖先node有三种状况：1.p，q分别存在于node的左右子树中 2.p为node，q在p的左子树或者右子树中 3.q为node，p在q的左子树或者右子树中。因此我们从root开始向下递归，寻找节点p和q

**递归分析：**

终止条件：递归到底时，返回null；找到p或q时返回p或q（递归到底时返回好理解，但是找到p或q就终止递归的原因是：存在两种情况：以找到了p为例，情况1:q在p的子树中，因此p就是两节点的最近公共祖先因此没有往下递归的必要，往上返回p；情况2:q不在p的子树中，那么也没有往下继续寻找的必要了）

递归过程：对于递归中的每一个节点node分别有几种情况：1.left和right都不为空：p，q分别在其的左右子树中，那么此时node就是p，q的最近祖先节点；2.left和right一边为空：node不是最近的祖先节点，此时分为两种情况：a.最近的祖先节点已经在不为空的那边的子树中找到，往上抛 b.最近的祖先节点还没找到但是找到了p，q其中之一，往上抛；因此在一边为空的情况下只需要返回不为空的那边就满足意义

特殊情况分析：再例如q在p的子树中的这种情况里，不需要通过找到left right都不为空的节点，只要找到p然后一直往上抛即为答案
