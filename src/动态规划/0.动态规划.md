# 动态规划

#### 1.对于子问题的定义，什么时候定义为“前n个元素”什么时候定义为“以nums[n]结尾的前n个元素”

1. 以某个元素结尾的子问题

**适用场景：连续约束**

- 比如 **乘积最大子数组** 或 **最长递增子序列（LIS）**。
- 为什么？因为问题有“连续性”或者“以当前元素为末尾”的约束。
  - 乘积最大子数组必须 **连续**，所以当前最大乘积和前面连续子数组紧密相关。
  - LIS 如果定义 dp[i] 为前 n 个元素的最大递增子序列长度，你就**不知道这个序列是否以 nums[i] 结尾**，无法正确更新 dp。
- 换句话说，你需要知道“结尾元素是谁”，才能正确地考虑 **是否可以接上当前元素**。

2. 只考虑前 n 个元素

**适用场景：不受连续约束**

- 例如 **打家劫舍（House Robber）** 或 **背包问题**：

  - 打家劫舍：dp[i] = 前 i 个房子能抢到的最大金额。
  - 这里子问题的状态是前 n 个元素就够了，因为不需要记录“最后抢的是哪一间房子”才能转移。
  - 只要在状态转移时约束不违反连续性（抢了 i 就不能抢 i-1），你就可以直接用前 n 个元素来表示状态。
