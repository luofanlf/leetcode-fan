# [287. 寻找重复数](https://leetcode.cn/problems/find-the-duplicate-number/)

```java
class Solution {
    public int findDuplicate(int[] nums) {
        int slow = 0;
        int fast = 0;
        while(true){
            slow = nums[slow];
            fast = nums[fast];
            fast = nums[fast];
            if(slow == fast){
                int p = 0;
                while(p!=slow){
                    slow = nums[slow];
                    p = nums[p];
                }
                return p;
            }
        }
    }
}
```

因为数组的长度是n+1，但是只由1..n的数字构成，所以一定有重复的数字存在。

index: 0 1 2 3 4

value: 1 3 4 2 2

我们可以把题目转化一下，对于数组的每个数，其下标i为索引，值nums[i]则为指向的链表下个元素的索引，那么n+1个值但只有n个索引，所以一定有一个索引的元素被指向了两次以上，也就是链表一定有环，这个被指向了两次即环的入口位置的索引就是重复数，因此这道题目就转换成了[142. 环形链表 II](https://leetcode.cn/problems/linked-list-cycle-ii/)，首先定义快慢指针找到快慢指针的相遇点，然后可以推导出``a=n(b+c)+c``,也就是说两个slow指针，一个从头出发一个从相遇点出发，从头出发的走到环的入口，从相遇点的刚好从了c加环的n圈，两个指针会刚好在环的入口相遇，直接返回环的入口的索引，得到答案。
